# DINOv3 Swin 特征蒸馏修改记录

本文记录本项目中与 DINOv3 -> Swin 特征蒸馏相关的核心代码修改。modified by zhoujiwen：不记录 `CUDA_VISIBLE_DEVICES` 等显卡编号的反复调整，因为这属于运行环境选择，不属于模型或训练逻辑修改。

## 总体目标

modified by zhoujiwen：使用 DINOv3 teacher 对 Swin-Base student 做自监督特征蒸馏。由于官方预训练 checkpoint 只有 backbone、没有 `dino_head` 和 `ibot_head`，因此保留 DINOv3 原始 SSL 训练中的四类 loss 名义和组织结构，但把 DINO/iBOT 原本 head 后的 CE 蒸馏替换为 feature space cosine loss。

当前 loss 结构为：

```text
total_loss = dino_loss + ibot_loss + koleo_loss + gram_loss
```

## 核心架构

新增：

```text
dinov3/train/ssl_feature_distill_meta_arch.py
```

modified by zhoujiwen：新增 `SSLFeatureDistillationMetaArch`，不再沿用临时的 `FeatureDistillationMetaArch`。该类保留 DINO/iBOT/KoLeo/Gram 四类 loss 结构：

- `dino_loss`：使用 student/teacher CLS feature 的 cosine loss。
- `ibot_loss`：使用 masked patch feature 的 cosine loss。
- `koleo_loss`：使用 DINOv3 原始 `KoLeoLoss`。
- `gram_loss`：使用 DINOv3 原始 `GramLoss`，作用在 patch feature 上。

删除旧临时实现：

```text

scripts/run_swin_base_feature_distill.sh
```

训练入口注册：

```text
dinov3/train/train.py
```

modified by zhoujiwen：注册 `SSLFeatureDistillationMetaArch`，并将 CSV 字段调整为：

```text
epoch, step, total_loss, dino_loss, ibot_loss, koleo_loss, gram_loss, val_loss
```

## Teacher 与 Student

teacher 使用 DINOv3 ViT-7B：

```yaml
feature_distill:
  teacher_arch: vit_7b
  teacher_patch_size: 16
  teacher_checkpoint: /mnt/ht2_nas2/00-model/00-zhoujw/dinov3-swin-distill-7B-with-gram/checkpoints/dinov3_vit7b16_pretrain_lvd1689m-a955f4ea.pth
```

student 使用 Swin-Base：

```yaml
student:
  arch: swin_base
  patch_size: 4
```

modified by zhoujiwen：ViT-7B teacher 输出维度为 4096，Swin-Base student 输出维度为 1024，因此新增 student feature projection：

```python
nn.Sequential(
    nn.Linear(student_dim, teacher_dim, bias=False),
    nn.LayerNorm(teacher_dim),
)
```

该 projection 仅用于 DINO/iBOT/Gram 的 feature 对齐；KoLeo 仍作用在投影前的 Swin CLS feature 上。

## 超大图过滤

modified by zhoujiwen：训练集中存在少量超大图，会触发 PIL 的 `DecompressionBombError`。在 `FlatJPGDataset` 初始化阶段过滤：

```text
width * height > 89478485
```

实现位置：

```text
dinov3/data/datasets/flat_jpg.py
```

使用：

```python
Image.open(path).size
```

只读取图片尺寸，同时过滤损坏图片，避免 DataLoader 在训练中断。modified by zhoujiwen：尺寸探测阶段使用 `warnings.catch_warnings()` 静默 `DecompressionBombWarning`，因为 PIL 会在返回 `img.size` 前先打印 warning；静默 warning 不改变过滤逻辑。

## 训练配置

modified by zhoujiwen：当前训练集 jpg 数量为 990848，2 卡训练、每卡 batch size 为 64，因此：

```text
global_batch_size = 64 * 2 = 128
OFFICIAL_EPOCH_LENGTH = ceil(990848 / 128) = 7741
```

配置：

```yaml
train:
  batch_size_per_gpu: 64
  OFFICIAL_EPOCH_LENGTH: 7741

optim:
  epochs: 3000

feature_distill:
  val_period_iterations: 7741
  early_stopping_patience: -1
```

modified by zhoujiwen：`early_stopping_patience=-1` 表示禁用 early stopping，对应训练代码中仅当 `early_stopping_patience >= 0` 时才触发 early stop。

## Checkpoint

modified by zhoujiwen：`OFFICIAL_EPOCH_LENGTH` 保持 7741，但 checkpoint 保存周期不再绑定 epoch，避免长时间训练中断后无恢复点：

```yaml
checkpointing:
  period: 100
  max_to_keep: 5
  keep_every: 22500
```

当前训练使用 FSDP/DCP checkpoint，保存目录中可能包含：

```text
_0_0.distcp
_1_0.distcp
.metadata
```

这是分布式 checkpoint，不是普通 `.pth` 单文件。加载时应加载整个 checkpoint 目录。
